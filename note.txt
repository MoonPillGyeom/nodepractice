동기와 비동기, 블로킹과 논블로킹

동기와 비동기 : 함수가 바로 return되는지 여부
블로킹과 논블로킹 : 백그라운드 작업 완료 여부

----
노드에서는 동기-블로킹 or 비동기-논블로킹 방식이 대부분

동기-블로킹 : 백그라운드 작업 완료 여부를 계속 확인하며, 호출한 함수가 바로 return되지 않고 백그라운드 작업이 끝나야 return
비동기-논블로킹 : 호출한 함수가 바로 return되어 다음 작업으로 넘어가고, 백그라운드 작업 완료 여부는 신경 쓰지 않고 나중에 백그라운드가 알림을 줄 때 처리

버퍼와 스트림

readFile 메서드를 사용할 때 읽었던 파일이 버퍼 형식으로 출력.
노드는 파일을 읽을 때 메모리에 파일 크기만큼 공간을 마련해두며, 파일 데이터를 메모리에 저장한 뒤 사용자가 조작할 수 있도록 해줌
메모리에 저장된 데이터가 바로 버퍼
버퍼를 직접 다루는 클래스 Buffer
