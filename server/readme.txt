쿠키와 세션 이해하기

클라이언트에서 보내는 요청은 한 가지 큰 단점이 있다.
누가 요청을 보내는지 모른다는 것 
요청을 보내는 IP주소나 브라우저의 정보를 받아올 순 있다.
하지만 여러 컴퓨터가 공통으로 IP주소를 가지거나, 한 컴퓨터를 여러 사람이 사용할 수도 있음 ex)PC방, 프린터가게

로그인을 구현해야함

쿠키는 name=pillgyeom 같이 단순한 '키-값'

서버로부터 쿠키가 오면 웹 브라우저는 쿠키를 저장해두었다가 요청할때마다 쿠키를 동봉해서 보냄
해당 쿠키를 읽어서 사용자가 누구인지 파악

브라우저는 쿠키가 있다면 자동으로 동봉 해줌
서버에서 브라우저로 쿠키를 보낼 때만 코드를 작성하여 처리하면 됨

쿠키는 요청과응답 헤더(header)에 저장됨
요청과응답은 각각 헤더와 본문(body)를 가짐
